# 验收标准
第二里程碑的验收标准要严格，第二里程碑之后就是铺量了，这个版本是可以表现游戏核心机制的。

## 主要验证项目

### 手机操作：不能感觉很难受，操作失误要达到一个标准。例如：失误率要低于20%
- **触控响应时间**：点击操作响应延迟不超过100ms
- **误触防护机制**：
  - 建造模式与选择模式有明确区分，避免误触
  - 单位选择框选有最小尺寸限制（不低于20像素）
  - 建造预览状态有明确视觉提示（半透明建筑模型+颜色区分）
- **操作精度要求**：
  - 建筑放置精度误差不超过1个网格单位
  - 单位选择准确率不低于90%
  - 移动指令准确率不低于90%
- **手势识别标准**：
  - 拖动操作触发阈值：手指移动超过10像素且时间超过200ms
  - 点击与双击区分：双击间隔小于300ms且位移小于5像素
- **用户测试指标**：
  - 新手用户完成基础操作（建造发电厂→采矿场→坦克工厂→生产坦克）时间不超过3分钟
  - 操作失误率（如误选单位、误放建筑）低于20%
  - 用户满意度评分不低于4.0（满分5.0）

### 地图寻路，坦克群里移动都要解决
- **寻路算法性能**：
  - 单个单位寻路计算时间不超过10ms
  - 10个单位同时寻路计算时间不超过50ms
  - 寻路成功率100%
- **群体移动优化**：
  - 10辆坦克集群移动不出现明显重叠或卡顿
  - 群体移动路径规划考虑单位半径，保持合理间距
  - 集群移动时前导坦克优先通过狭窄通道
- **路径平滑处理**：
  - 路径拐点进行平滑处理，避免坦克直角转弯
  - 移动过程中坦克朝向与移动方向保持一致
- **障碍物处理**：
  - 坦克能正确识别并绕过静态障碍物（建筑、岩石等）
  - 坦克能动态避让其他移动单位
  - 寻路考虑坦克半径，避免贴近障碍物边缘行走

### 手机操作得流畅
- **帧率标准**：
  - 低端设备（Android 6.0以上）稳定运行30FPS以上
  - 高端设备稳定运行60FPS
  - 界面切换动画帧率不低于30FPS
- **内存管理**：
  - 运行内存占用控制在200MB以内
  - 无明显内存泄漏，连续游戏30分钟内存增长不超过10%
- **加载时间**：
  - 首次启动时间不超过5秒
  - 场景切换时间不超过2秒
  - 建筑建造完成到生成单位延迟不超过1秒
- **网络延迟容忍**：
  - 网络延迟在200ms以内时，操作无明显卡顿
  - 网络延迟在500ms时，可通过预测机制保证基本可玩性

### 基本的音效
- **音效完整性**：
  - 建造建筑：开始建造、建造完成、电力不足警告
  - 单位操作：选择、移动、攻击、受损、死亡
  - 界面操作：按钮点击、菜单打开/关闭
  - 游戏状态：胜利、失败、资源采集
- **音频质量标准**：
  - 音效采样率不低于44.1kHz
  - 格式统一使用压缩格式（mp3或ogg）
  - 同类音效有明显区分度
- **音频性能**：
  - 同时播放音效数量不少于8个
  - 音效触发延迟不超过50ms
  - 音频资源占用总大小控制在10MB以内

### 不能有明显的BUG
- **功能完整性**：
  - 所有设计功能100%实现
  - 游戏规则逻辑正确（电力系统、经济系统、战斗系统）
  - 胜负判定准确无误
- **稳定性要求**：
  - 连续游戏1小时无崩溃
  - 各种边界条件下无死机或严重卡顿
  - 异常输入处理得当，不导致程序异常
- **兼容性测试**：
  - Android 6.0及以上系统正常运行
  - iOS系统正常运行
  - 主流分辨率适配正常（720p、1080p、2K）
- **数据一致性**：
  - 帧同步机制保证两端状态完全一致
  - 无因网络延迟导致的游戏状态不同步
  - 存档数据读写正确，无数据丢失
