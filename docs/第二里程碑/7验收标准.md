# 验收标准

### 🧭 寻路移动验收标准

寻路移动的体验，核心目标是让玩家感觉智能、自然且可靠。你可以从以下几个维度进行测试和验收：

| 验收维度 | 具体场景/指标 | 验收标准 |
| :--- | :--- | :--- |
| **基础路径查找** | 简单开阔场景、复杂障碍场景 | 角色能自动计算并行走**最短/最优路径**，无长时间卡顿或无法到达的情况。 |
| **动态障碍应对** | 路径上突然出现动态障碍物 | 角色能**实时重新计算路径**并有效绕行，无明显顿挫感。 |
| **复杂地形适应性** | 不同移动代价的地形（如沼泽、平地） | 角色能根据算法（如A*）**优先选择代价更低的路径**，而并非单纯最短的直线。 |
| **群体移动表现** | 多个单位同时向同一区域移动 | 单位间有**基础的碰撞避免**，不会大量重叠或相互卡死，移动轨迹相对自然。 |

### 🎮 操作手感验收标准

操作手感更侧重于玩家输入与游戏反馈之间那种微妙而直接的感受，追求高响应性、高准确性和高容错性。

| 验收维度 | 具体场景/指标 | 验收标准 |
| :--- | :--- | :--- |
| **响应与延迟** | 角色移动、镜头旋转、开枪射击等 | 操作反馈**感觉不到延迟**，或延迟在可接受范围内（如低于50毫秒）。 |
| **输入精准与平滑** | 不同幅度和速度的滑动操作 | 镜头转动速度能**自适应操作力度**，小幅度微调精准，快速转身平滑。可通过**输入归一化处理**确保不同设备体验一致。 |
