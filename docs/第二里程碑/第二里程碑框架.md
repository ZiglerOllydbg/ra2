# RTS手游第二里程碑 - 最小化设计框架

## 🎯 核心定位
**最小可行RTS经济循环**：在最小功能集基础上实现完整的"采集-建造-生产-战斗"循环

## 📋 整体架构

### 1. 游戏系统层
```
经济系统
├── 资金管理（采集+消耗）
├── 电力管理（供应+需求）
└── 建造队列

战斗系统  
├── 单位移动控制
├── 自动攻击机制
└── 胜负判定逻辑

同步系统
├── 帧同步核心
├── 状态一致性
└── 手机网络优化
```

### 2. 内容层
```
建筑体系（4种）
├── 主营地（核心）
├── 发电厂（电力）
├── 采矿场（经济）  
└── 坦克工厂（军事）

单位体系（1种）
└── 基础坦克（移动+攻击）

资源体系（2种）
├── 资金（通用货币）
└── 电力（建造限制）
```

### 3. 操作层
```
界面布局
├── 顶部：资源信息+小地图
├── 中央：游戏主视角
└── 底部：操作按钮区

核心操作
├── 建造：按钮→选择→放置
├── 选择：点击/框选单位
├── 移动：选中→点击目标
└── 生产：建筑→队列管理
```

### 4. 资源层
```
3D模型（8个核心）
├── 4种建筑×2状态（完整+建造中）
└── 坦克×4状态（完整+移动+攻击+死亡）

特效（6个基础）
├── 建造特效×2
├── 采集特效×1
├── 战斗特效×2
└── UI特效×1

音频（14个必要）
├── 音效×12（界面+建造+战斗+状态）
└── 音乐×2（游戏+结算）

UI资源（15个元素）
├── 图标×7
├── 按钮×5
└── 界面组件×3
```

## 🎮 体验流程

### 标准游戏进程
```
开局（0-1分钟）
├── 初始资源：1000资金 + 3坦克
├── 首要目标：建造发电厂

发展期（1-3分钟）  
├── 建造采矿场→稳定收入
├── 建造坦克工厂→军事基础
└── 持续生产坦克

决战期（3分钟+）
├── 坦克数量积累
├── 寻找进攻机会
└── 攻击敌方主营地
```

### 操作学习曲线
```
初级（前2局）
├── 学会基础建造
├── 理解资源概念
└── 掌握单位移动

中级（3-5局）
├── 优化建造顺序
├── 管理电力平衡
└── 多单位控制

高级（5局+）
├── 时机把握
├── 资源调度优化
└── 战术策略运用
```
