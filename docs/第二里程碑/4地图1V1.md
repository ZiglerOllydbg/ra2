# 2人地图详细设计

## 🗺️ 基础布局

### 对称竞技场设计
```
地图尺寸：128×128单位
形状：正方形对称布局
主题：草原/荒地战场

[玩家A基地]━━━━━━[中央区域]━━━━━━[玩家B基地]
     │                              │
 [资源点A]                      [资源点B]
     │                              │
 [安全区A]                      [安全区B]
     │                              │
 [出生点A]                      [出生点B]
```

## 📍 关键位置坐标

### 玩家A区域（左下区域）
```
主营地位置：(20, 20)
建造区域：从(15,15)到(35,35)的矩形
起始资源点：(25, 40) - 储量3000
安全区边界：x≤45, y≤45
```

### 玩家B区域（右上区域）
```
主营地位置：(108, 108)  
建造区域：从(93,93)到(113,113)的矩形
起始资源点：(103, 88) - 储量3000
安全区边界：x≥83, y≥83
```

### 中央区域（核心战场）
```
中心点：(64, 64)
中立资源点1：(64, 50) - 储量5000
中立资源点2：(64, 78) - 储量5000
主要交战区：从(40,40)到(88,88)
```

## 🏞️ 地形特征

### 1. 基地区域（安全区：视觉安全，非无敌）
```
地形：完全平坦
纹理：浅绿色草地
特征：
- 明显的边界指示（颜色渐变）
- 充足的建筑空间（7×7逻辑网格，每个网格=3单位，实际覆盖21×21单位区域）
- 无任何障碍物
- 轻微的地面网格指示

【安全区核心设计目的】
- 节奏控制：确保早期阶段（0-2分钟）安全发展，无直接冲突
- 战略分界：形成清晰的安全发展区与交战区分界
- 防止速推：避免游戏在1分钟内结束，符合5-8分钟平均游戏时间
- 新手友好：提供明确的安全操作空间，降低初期压力
- 战术基础：为基地防御和进攻提供战略决策点
- 出生点保护：确保玩家初始单位不会立即受到攻击
```

### 2. 资源点设计
```
视觉特征：
- 金色发光粒子效果
- 特殊地面纹理（矿石纹理）
- 缓慢旋转的采集标志

功能区域：
- 半径3单位的圆形区域
- 明显高于周围地面
- 持续发光提示重要性
```

### 3. 中央交战区
```
地形变化：
- 轻微起伏，但不影响移动
- 几个小土坡提供轻微视野优势
- 随机分布的岩石作为掩护点

纹理混合：
- 主纹理：干燥土地
- 辅助纹理：岩石和草丛
- 路径痕迹：显示常用移动路线
```

## 🎨 视觉层次

#### 颜色编码系统
```
玩家A区域：蓝色调
- 地面：蓝绿色草地
- 建筑高亮：蓝色轮廓

玩家B区域：红色调  
- 地面：红褐色土地
- 建筑高亮：红色轮廓

中立区域：黄色/棕色
- 表示可争夺区域
- 资源点：金色高亮
```

#### 环境物件布置

##### 装饰性物件
```
草丛集群：
- 安全区边缘：稀疏草丛
- 中央区域：较密草丛，提供视觉遮挡
- 使用低面数模型（<100面）

岩石点缀：
- 小型岩石：随机分布在中央区域
- 大型岩石：作为战术掩护点（3-4个）
- 不可破坏，仅视觉和战术用途
```

##### 战略性物件
```
中央障碍物：
- 2个大型岩石组，形成天然通道
- 位置：(55,64)和(73,64)
- 迫使玩家选择绕行或狭窄通过

视野草丛：
- 中央区域4处高草丛
- 进入后单位半透明显示
- 提供伏击机会
```

## 🛣️ 路径设计

#### 主要通道
```
直接路线：
玩家A基地 → (45,45) → (64,64) → (83,83) → 玩家B基地

侧翼路线1（上方）：
玩家A → (35,55) → (55,75) → 玩家B

侧翼路线2（下方）：
玩家A → (55,35) → (75,55) → 玩家B
```

#### 战术要点
```
控制点1：中央资源点区域
- 视野开阔，易攻难守
- 同时控制两个富饶资源点

控制点2：通道狭窄处  
- 大型岩石形成的天然瓶颈
- 少量坦克即可防守

伏击点：高草丛区域
- 隐藏单位，突然袭击
- 适合以少打多
```

## ⚙️ 技术实现细节

#### 碰撞体设置
```
地面碰撞：单一平面碰撞体
建筑碰撞：2×2单位方形碰撞体
单位碰撞：半径0.5单位的球形碰撞体
障碍物碰撞：匹配模型形状的网格碰撞体
资源点碰撞：半径3单位的触发碰撞体
```

### 网格系统
```
- 基础单位：1网格 = 3游戏单位
- 建筑占用：每个建筑需2×2网格（6×6单位）
- 安全区实际：20×20单位 ≈ 6.67×6.67网格（理论最大7×7）
- 路径宽度：主要通道4网格（12单位）
- 网格对齐：建筑自动吸附到3单位网格
```

### 视野系统
```
基础视野范围：
- 建筑：8单位
- 坦克：10单位
- 主营地：12单位

视野阻挡：
- 高草丛：进入后敌方看不见
- 岩石：完全阻挡视野
- 地图边界：视野限制
```

### 小地图设计
```
小地图尺寸：屏幕左上角 120×120像素
显示内容：
- 玩家单位：蓝色点
- 敌方单位：红色点（在视野内）
- 建筑：方形图标
- 资源点：金色闪烁点
- 视野范围：渐变色显示
```

## 🎮 游戏性平衡

### 对称性保证
```
完全镜像设计：
- 相同的地形布局
- 相同的资源分布
- 相同的路径选择
- 相同的战术机会
- 对称的安全区设计（玩家A：x<45,y<45；玩家B：x>83,y>83）
```

### 战术选择支持
```
进攻选择：
1. 直接中路推进 - 快速但危险
2. 侧翼包抄 - 较慢但安全
3. 经济压制 - 控制中央资源点

防守选择：
1. 基地固守 - 利用建筑防御
2. 前沿防御 - 在通道处拦截
3. 机动防御 - 利用中央区域周旋
```

### 节奏控制
```
早期阶段（0-2分钟）：
- 安全区内安全发展，无直接冲突
- 专注于经济建设（发电厂、采矿场）
- 无法直接攻击对方基地

中期阶段（2-4分钟）：
- 小规模中央区域冲突
- 资源点争夺开始
- 安全区边界成为战略分界线

后期阶段（4分钟+）：
- 大规模坦克决战
- 基地攻防战（突破安全区）
- 主营地成为最终目标
```
